# Flow 2: HTML ダッシュボード自動生成

## フロー名
`[工数管理] HTML ダッシュボード生成`

## 概要
SharePoint の WorkHoursLog と Projects リストから全データを取得し、
Chart.js を使ったセルフコンテインド HTML ファイルを生成して
SharePoint の `WorkHoursDashboard` ドキュメントライブラリに保存する。

---

## トリガー

**コネクタ**: SharePoint
**アクション**: `項目が作成または変更されたとき`
- サイトのアドレス: `[YOUR_SHAREPOINT_SITE_URL]`
- リスト名: `WorkHoursLog`

> **ポイント**: Flow 1 が工数を新規登録、Flow 3 が工数を更新/Cancelledに変更 → いずれも
> WorkHoursLog の項目が変更されるため、このトリガーで**自動的に**フロー発火する。
> Flow 1/3 から別途呼び出す必要はない。
>
> **注意事項**:
> - SharePoint トリガーのポーリング間隔は**約1〜5分**。即時反映ではないが実用上問題ない。
> - 短時間に複数の予定が登録/変更されると連続発火するが、毎回最新の全データで HTML を
>   再生成するため、最終結果は常に正しい。
> - Power Automate の「トリガー条件」は設定しない（全変更で再生成する）。

---

## ステップ一覧

### Step 1: WorkHoursLog 全件取得

WorkHoursLog リストの全レコードを取得する。

#### 1-1. アクションの追加
1. トリガーの下の **＋** ボタン → **アクションの追加** をクリック
2. 検索ボックスに `SharePoint` と入力
3. **SharePoint** コネクタを選択
4. アクション一覧から **「複数の項目の取得」** をクリック
5. アクション名を `WorkHoursLog取得` に変更する
   （アクション右上の `…` → **名前の変更**）

#### 1-2. パラメータの設定
| パラメータ | 設定値 | 操作方法 |
|-----------|--------|---------|
| サイトのアドレス | `[YOUR_SHAREPOINT_SITE_URL]` | ドロップダウンから自分の SharePoint サイトを選択 |
| リスト名 | `WorkHoursLog` | ドロップダウンから選択 |

#### 1-3. 詳細オプションの設定
「詳細オプションを表示する」をクリックして、以下を入力:

| パラメータ | 設定値 | 説明 |
|-----------|--------|------|
| フィルター クエリ | （空欄のまま） | 全件取得する |
| 並び替え順序 | `StartDateTime desc` | 新しい予定が先頭に来る |
| 上から順に取得 | `5000` | 取得上限。既定値だと100件で切れるため明示指定する |
| 列で制限するビュー | （空欄のまま） | 使用しない |

> **⚠️ 「上から順に取得」を必ず設定すること。**
> 既定値（空欄）だと上限 100 件しか取得されず、101 件目以降の工数ログが
> ダッシュボードに表示されなくなる。

#### 2-2. パラメータの設定
| パラメータ | 設定値 | 操作方法 |
|-----------|--------|---------|
| サイトのアドレス | `[YOUR_SHAREPOINT_SITE_URL]` | Step 1 と同じサイトを選択 |
| リスト名 | `Projects` | ドロップダウンから選択 |

#### 2-3. 詳細オプションの設定
「詳細オプションを表示する」をクリック:

| パラメータ | 設定値 | 説明 |
|-----------|--------|------|
| フィルター クエリ | （空欄のまま） | 全件取得（非アクティブも含めて HTML に渡す） |
| 並び替え順序 | （空欄のまま） | 順序不問 |

> **ポイント**: Step 1 と Step 2 は互いに依存しないため、Power Automate 上で
> **並列分岐（Parallel branch）** に配置すると実行が少し速くなる。
> ただし必須ではないので、単純に上から並べても問題ない。

### Step 3: 必要な列だけを取り出す（Select アクション）

SharePoint の「複数の項目の取得」は、各レコードに大量の管理情報（作成者、更新日時、
内部ID など）を含んだ形で返す。HTML テンプレートに渡すには**必要な列だけ**を
取り出して整形する必要がある。これを行うのが **Select** アクション。

> **Select アクションとは？**
> 配列（リスト）の各要素から、指定したキー名と値だけを取り出して
> 新しい配列を作るアクション。Excel の「必要な列だけコピーして新しい表を作る」イメージ。

#### 3-1. WorkHoursLog 用の Select を追加する

##### アクションの追加
1. Step 2（Projects取得）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「Select」** と入力（日本語で「選択」でもOK）
3. **「データ操作」** コネクタの **「選択」** アクションをクリック
4. アクション名を `MappedLogs` に変更する
   （アクション右上の `…` → **名前の変更**）

##### 「開始」欄の設定
1. **「開始」** 欄をクリック
2. 入力欄右側の **稲妻アイコン**（動的なコンテンツ）ではなく、
   その隣の **「式」** タブをクリック
3. 以下の式を入力して **OK** をクリック:
   ```
   body('WorkHoursLog取得')?['value']
   ```
   > この式は「Step 1 で取得した WorkHoursLog の全レコード配列」を意味する。
   > `'WorkHoursLog取得'` は Step 1 で設定したアクション名と**完全一致**が必要。

##### 「マップ」欄の設定
1. **「マップ」** の右にあるトグルをクリックして **「テキストモード」** に切り替える
   （トグルが「T」のアイコンに変わる）

   > **⚠️ 重要**: テキストモードに切り替えないと、キー名と値を自由に入力できない。

2. 左側（キー）と右側（値）を、1行ずつ以下のように入力する:

| 左側に入力するキー名 | 右側に入力する式（「式」タブで入力） |
|-------------------|--------------------------------------|
| `Title` | `item()?['Title']` |
| `EventId` | `item()?['EventId']` |
| `StartDateTime` | `item()?['StartDateTime']` |
| `EndDateTime` | `item()?['EndDateTime']` |
| `DurationHours` | `item()?['DurationHours']` |
| `ProjectCode` | `item()?['ProjectCode']` |
| `ProjectName` | `item()?['ProjectName']` |
| `Status` | `item()?['Status']?['Value']` |

> **「マップ」の行を増やすには？**
> 1行目を入力した後、左下の **「+ 新しい項目の追加」** ボタンをクリックすると行が増える。
> 上記の 8 列分、繰り返し追加する。

> **右側の値の入力方法（重要）**:
> - 右側の入力欄をクリック → **「式」タブ** を選択
> - 式を入力して **OK** をクリック
> - 「動的なコンテンツ」タブではなく、必ず **「式」タブ** を使うこと

> **⚠️ Status の式について:**
> SharePoint で Status を**選択肢列（Choice）** として作成した場合、
> SharePoint は `{"Value": "Active"}` のようなオブジェクトでデータを返す。
> そのため `item()?['Status']?['Value']` で中の文字列を取り出す。
>
> もし Status を**テキスト列**として作成した場合は `item()?['Status']` でよい。
> 判断がつかない場合は、一度フローを実行してから実行履歴で出力を確認する。

---

#### 3-2. Projects 用の Select を追加する

##### アクションの追加
1. Step 3-1（MappedLogs）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「Select」** と入力
3. **「データ操作」** コネクタの **「選択」** アクションをクリック
4. アクション名を `MappedProjects` に変更する

##### 「開始」欄の設定
1. **「開始」** 欄 → **「式」** タブ → 以下を入力して OK:
   ```
   body('Projects取得')?['value']
   ```
   > `'Projects取得'` は Step 2 で設定したアクション名。

##### 「マップ」欄の設定
1. **テキストモード** に切り替える
2. 以下のように入力:

| 左側に入力するキー名 | 右側に入力する式（「式」タブで入力） |
|-------------------|--------------------------------------|
| `Title` | `item()?['Title']` |
| `ProjectCode` | `item()?['ProjectCode']` |
| `Color` | `item()?['Color']` |
| `PlannedHours` | `item()?['PlannedHours']` |
| `IsActive` | `item()?['IsActive']` |

### Step 4: JSON テキストに変換する（Compose アクション）

Step 3 で作った配列データを、HTML に埋め込める**テキスト形式の JSON 文字列**に変換する。

> **なぜこのステップが必要？**
> Step 3 の Select アクションの出力はPower Automate 内部の「配列オブジェクト」であり、
> そのままでは HTML のテンプレートに文字列として埋め込めない。
> `string()` 関数で `[{"Title":"...","EventId":"..."},...]` のような
> テキスト形式に変換する。

#### 4-1. WorkHoursLog の JSON 変換

##### アクションの追加
1. Step 3-2（MappedProjects）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「Compose」** と入力（日本語で「作成」でもOK）
3. **「データ操作」** コネクタの **「作成」** アクションをクリック
4. アクション名を `LogsJson` に変更する

##### 「入力」欄の設定
1. **「入力」** 欄をクリック → **「式」タブ** を選択
2. 以下の式を入力して **OK**:
   ```
   string(body('MappedLogs'))
   ```
   > `'MappedLogs'` は Step 3-1 で設定したアクション名。

---

#### 4-2. Projects の JSON 変換

##### アクションの追加
1. Step 4-1（LogsJson）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「Compose」** と入力
3. **「データ操作」** コネクタの **「作成」** アクションをクリック
4. アクション名を `ProjectsJson` に変更する

##### 「入力」欄の設定
1. **「入力」** 欄 → **「式」タブ** → 以下を入力して OK:
   ```
   string(body('MappedProjects'))
   ```

> **ここまでのフロー全体像**:
> ```
> [トリガー] WorkHoursLog の項目が作成/変更されたとき
>     ↓
> [複数の項目の取得] WorkHoursLog取得（全件取得）
>     ↓
> [複数の項目の取得] Projects取得（全件取得）
>     ↓
> [選択] MappedLogs（必要な列だけ取り出す）
>     ↓
> [選択] MappedProjects（必要な列だけ取り出す）
>     ↓
> [作成] LogsJson（テキストに変換）
>     ↓
> [作成] ProjectsJson（テキストに変換）
>     ↓
> （Step 5 以降へ続く）
> ```

### Step 5: HTML テンプレートを読み込む

SharePoint にアップロード済みの HTML テンプレートファイルを読み込み、
テキスト（文字列）として変数に格納する。

#### 5-1. ファイルのコンテンツを取得する

##### アクションの追加
1. Step 4-2（ProjectsJson）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「SharePoint」** と入力
3. **SharePoint** コネクタの **「ファイルのコンテンツの取得」** をクリック
4. アクション名を `テンプレート読み込み` に変更する

##### パラメータの設定
| パラメータ | 設定値 |
|-----------|--------|
| サイトのアドレス | ドロップダウンから自分の SharePoint サイトを選択 |
| ファイル識別子 | 右側のフォルダアイコンをクリックし、`WorkHoursDashboard` → `_template` → `index.html` を選択 |

> フォルダアイコンからファイルを選べない場合は、直接
> `/WorkHoursDashboard/_template/index.html` と入力してもよい。

---

#### 5-2. テキスト形式に変換して変数に格納する

SharePoint から取得したファイルは**バイナリ（Base64）形式**で返される。
これをテキスト文字列に変換して変数に保存する。

##### アクションの追加
1. **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「変数」** と入力
3. **「変数」** コネクタの **「変数を初期化する」** をクリック

##### パラメータの設定
| パラメータ | 設定値 |
|-----------|--------|
| 名前 | `HTMLTemplate` |
| 型 | ドロップダウンから **「文字列」** を選択 |
| 値 | **「式」タブ** で以下を入力して OK: |

```
base64ToString(body('テンプレート読み込み')?['$content'])
```

> **この式の意味**:
> `body('テンプレート読み込み')` → Step 5-1 で取得したファイルデータ
> `?['$content']` → その中のファイル本体（Base64 形式）
> `base64ToString(...)` → Base64 を人間が読めるテキストに変換

---

### Step 6: テンプレートにデータを埋め込む

HTML テンプレートの中にある**目印（プレースホルダー）** を、
Step 4 で作った実データの JSON テキストに**置換**する。

> **仕組みのイメージ**:
> テンプレートの中にはこう書いてある:
> ```javascript
> const WORK_LOGS = /* LOGS_DATA_JSON */[];
> const PROJECTS = /* PROJECTS_DATA_JSON */[];
> const GENERATED_AT = "/* GENERATED_AT */";
> ```
> `/* LOGS_DATA_JSON */` の部分を実際のデータに置き換えると:
> ```javascript
> const WORK_LOGS = [{"Title":"要件定義MTG","DurationHours":2,...},...][];
> const PROJECTS = [{"Title":"プロジェクトA","Color":"#FF6384",...},...][];
> const GENERATED_AT = "2026-03-15T12:00:00Z";
> ```
> これでブラウザがデータを読み込めるようになる。

#### アクションの追加
1. Step 5-2 の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「Compose」** と入力（日本語で「作成」でもOK）
3. **「データ操作」** コネクタの **「作成」** をクリック
4. アクション名を `GeneratedHTML` に変更する

#### 「入力」欄の設定
1. **「入力」** 欄をクリック → **「式」タブ** を選択
2. 以下の式を**そのまま**コピー＆ペーストして **OK**:

```
replace(replace(replace(variables('HTMLTemplate'),'/* LOGS_DATA_JSON */',outputs('LogsJson')),'/* PROJECTS_DATA_JSON */',outputs('ProjectsJson')),'/* GENERATED_AT */',utcNow())
```

> **この式の意味（3段階の置換）**:
>
> | 順番 | 探す文字列（目印） | 置き換える内容 |
> |------|-------------------|--------------|
> | 1 | `/* LOGS_DATA_JSON */` | 工数ログの JSON（Step 4-1 の出力） |
> | 2 | `/* PROJECTS_DATA_JSON */` | プロジェクトの JSON（Step 4-2 の出力） |
> | 3 | `/* GENERATED_AT */` | 現在日時（`utcNow()`） |
>
> `replace(元の文字列, 探す文字列, 置き換え文字列)` を 3 回入れ子にしている。

> **⚠️ 式の入力が長い場合**: Power Automate の式入力欄は狭いが、
> 上記の式をメモ帳等にコピーしてから貼り付けると間違いが少ない。

### Step 7: SharePoint にファイルを保存する

Step 6 で完成した HTML を、SharePoint のドキュメントライブラリに保存する。

SharePoint の「ファイルの作成」アクションは、同名ファイルが既に存在すると
**別名（index1.html 等）で保存されてしまう**。
これを避けるため、**既存ファイルを削除してから新規作成**する。

#### 7-1. 既存ファイルを削除する（存在する場合のみ）

##### アクションの追加
1. Step 6（GeneratedHTML）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「SharePoint」** と入力
3. **SharePoint** コネクタの **「ファイルの削除」** をクリック
4. アクション名を `既存ファイル削除` に変更する

##### パラメータの設定
| パラメータ | 設定値 |
|-----------|--------|
| サイトのアドレス | ドロップダウンから自分の SharePoint サイトを選択 |
| ファイル識別子 | `/WorkHoursDashboard/index.html` と直接入力 |

> フォルダアイコンからファイルを選んでもよいが、
> まだファイルが存在しない初回セットアップ時は直接入力する。

##### エラー時の動作を設定する（初回はファイルが存在しないため）
1. アクション右上の `…` → **「実行条件の構成」** をクリック
2. **「成功」** にチェックが入っていることを確認
3. **「失敗」** にもチェックを入れる → **完了**

> **なぜ「失敗」にもチェックを入れるのか？**
> 初回実行時や何らかの理由でファイルが存在しない場合、「ファイルの削除」は
> エラーになる。「失敗」にもチェックを入れると、エラーが起きてもフローが
> 停止せず次のステップに進む。

---

#### 7-2. 新しい HTML ファイルを作成する

##### アクションの追加
1. Step 7-1（既存ファイル削除）の下の **＋** ボタン → **アクションの追加**
2. 検索ボックスに **「SharePoint」** と入力
3. **SharePoint** コネクタの **「ファイルの作成」** をクリック
4. アクション名を `HTMLファイル作成` に変更する

##### パラメータの設定
| パラメータ | 設定値 | 操作方法 |
|-----------|--------|---------|
| サイトのアドレス | `[YOUR_SHAREPOINT_SITE_URL]` | ドロップダウンから自分の SharePoint サイトを選択 |
| フォルダーのパス | `/WorkHoursDashboard` | 右側のフォルダアイコンから `WorkHoursDashboard` を選択 |
| ファイル名 | `index.html` | そのまま入力 |
| ファイル コンテンツ | Step 6 の出力 | 下記参照 |

##### 「ファイル コンテンツ」の入力方法
1. **「ファイル コンテンツ」** 欄をクリック
2. **「式」タブ** を選択
3. 以下を入力して **OK**:
   ```
   outputs('GeneratedHTML')
   ```
   > `'GeneratedHTML'` は Step 6 で設定したアクション名。

##### 実行条件の設定（削除が成功しても失敗しても実行する）
1. アクション右上の `…` → **「実行条件の構成」** をクリック
2. **「成功」** にチェック
3. **「失敗」** にもチェック → **完了**

> **なぜ「失敗」にもチェックを入れるのか？**
> Step 7-1 でファイルが存在しなかった場合、削除が「失敗」になるが、
> ファイル作成は問題なく実行したい。両方にチェックを入れることで、
> 「削除が成功しても失敗しても、必ずファイルを作成する」という動作になる。

> **フロー上の見た目**:
> ```
> [ファイルの削除] 既存ファイル削除
>     ↓（成功でも失敗でも次へ）
> [ファイルの作成] HTMLファイル作成
> ```
>
> - 初回実行時: 削除が失敗（ファイルなし）→ 新規作成される
> - 2回目以降: 削除が成功 → 新規作成される（＝上書きと同じ結果）
> 初回実行時: 「既存ファイル確認」が失敗 → 「ファイル新規作成」が実行される
> 2回目以降: 「既存ファイル確認」が成功 → 「ファイル上書き」が実行される

## フロー完成後の全体像

```
[トリガー] WorkHoursLog の項目が作成/変更されたとき
    ↓
[複数の項目の取得] WorkHoursLog取得
    ↓
[複数の項目の取得] Projects取得
    ↓
[選択] MappedLogs（必要な列だけ取り出す）
    ↓
[選択] MappedProjects（必要な列だけ取り出す）
    ↓
[作成] LogsJson（テキストに変換）
    ↓
[作成] ProjectsJson（テキストに変換）
    ↓
[ファイルのコンテンツの取得] テンプレート読み込み
    ↓
[変数を初期化する] HTMLTemplate（テキストに変換）
    ↓
[作成] GeneratedHTML（データを埋め込む）
    ↓
[ファイルの削除] 既存ファイル削除（なければスキップ）
    ↓
[ファイルの作成] HTMLファイル作成
```

## テスト手順

1. SharePoint の `WorkHoursDashboard` に `_template` フォルダを作成し、
   `index.html`（テンプレート）をアップロードする
2. WorkHoursLog リストに任意のテスト行を追加する（これが Flow 2 のトリガーになる）
3. 数分後、`WorkHoursDashboard` フォルダに `index.html` が作成されることを確認
4. `index.html` をダウンロードしてブラウザで開く
5. 円グラフ・棒グラフ・テーブルが表示されることを確認
6. Cancelled レコードが集計から除外されていることを確認
